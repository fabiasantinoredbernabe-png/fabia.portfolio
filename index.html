Midterm Activity 3: ATM with Transfer Feature

Lesson 3

Program Requirements

1. Create a Java program named ATMWithdrawalLimit.

2. Assume the initial balance is 9,000 pesos.

===== ATM MENU ===== 1. Check Balance   2. Deposit   3. Withdraw   4. Exit

Withdrawal Rules: Maximum withdrawal per transaction: 3,000 pesos. If exceeded, display: Withdrawal Limit Exceeded.

Use: if-else statements, switch statement, do-while loop.

Reflections

  • In this lesson, I learned how control flow statements like if-else, switch, and do-while loops help manage the behavior of a Java program. The if-else statement allows me to make decisions based on conditions, while the switch statement provides a cleaner way to handle multiple possible values of a variable. I also learned that the do-while loop ensures that a block of code runs at least once before checking the condition, which is useful in certain situations. Overall, these concepts improved my understanding of how to control the flow of execution and make programs more interactive and logical.

Midterm Activity 4: Student Library Borrowing System with Penalty Counter

Lesson 4

Objective: Create a Java program that simulates a student payment system using decision structures, loops, and user-defined validation.

1. Create a Java program named StudentPaymentSystemLastName.

2. Ask the user to input: Student Name, Student ID, Total Tuition Fee.

3. Display the ===== PAYMENT MENU ===== with options: 1. Pay Tuition   2. Check Balance   3. Apply Discount   4. Exit

General Rules: Use switch statement for menu options, do-while loop until Exit, nested if statements for validation, and a counter variable to track transactions.

Additional Conditions: Prevent negative balance, prevent multiple discount applications, show "No remaining balance" if fully paid.

Reflections

  • In this lesson, I learned how to combine different control structures to create a more functional and user-friendly payment system program. Using a switch statement to process menu options made the program more organized and easier to manage. The do-while loop helped ensure that the program continues running until the user chooses to exit, making it more interactive. I also understood how nested if statements are useful for validating user input and handling different conditions properly. Additionally, using a counter variable allowed me to track the number of transactions, which adds more control and monitoring to the program. Overall, this lesson strengthened my ability to design structured and efficient programs.

Midterm Activity 5: Personal Expense Tracker

Lesson 5

Instructions:

1. Create a Java program named ExpenseTrackerLastName.

2. Implement the following methods: a void method for the program title; a non-void method to calculate total expenses (food, transportation, other); a non-void method to check if total exceeds budget; a void method to display total and budget status.

3. In the main method, call all created methods and display the final output clearly.

4. Use appropriate method names, parameters, and return types.

5. Add at least TWO enhancements: accept user input, add another expense category, compute remaining budget, or display a personalized message.

6. Ensure your program compiles without errors and produces correct output.

Reflections

  • I learned about methods in Java, including the difference between void and non-void methods. A void method performs a task without returning a value, while a non-void method returns a result that can be used in other parts of the program. I also understood the importance of method names, which should clearly describe the purpose of the method, as well as parameters that allow data to be passed into a method. Learning about return types helped me see how methods can produce outputs and make programs more modular and reusable. Overall, this lesson improved my understanding of how to organize code using methods in Object-Oriented Programming.
